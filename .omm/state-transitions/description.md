The embedder-model provisioning lifecycle (EmbedderModelStatus in BipboxCore/TextEmbedder.swift, driven by ModelProvisioningCoordinator in BipboxApplication.swift). On launch: if the marker file exists the coordinator silent-loads the cached model (banner-free path to ready); otherwise the UI shows the explicit Download banner — the model is NEVER fetched silently, so first start makes the ~600MB download obvious. ready fires onBecameReady exactly once -> EmbeddingBackfillService embeds vectorless items -> WorkspaceModel.recomputeClusters renders topics.
