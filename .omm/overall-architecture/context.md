The architecture exists to keep the domain testable without macOS frameworks: 386 headless tests run via swift test against the same composition root the app uses. The MLX dependency is isolated in its own target because swift-build binaries cannot load MLX's Metal library — only Xcode-built .app bundles include default.metallib — so everything except BipboxApp must stay runnable without it (tests and harness default to NLEmbeddingTextEmbedder via the embedderFactory injection point).
