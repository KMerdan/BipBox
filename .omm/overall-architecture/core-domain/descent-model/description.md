The folder-vs-project decision, ported from the validated experiment. FilesystemDescender classifies every directory: marker folders (.git, Package.swift, package.json, Cargo.toml, ...) are one PROJECT unit and descent stops; workspace monorepos surface member projects; bundle suffixes (.app, .photoslibrary, ...) are opaque; a non-project folder with no project children collapses into one COLLECTION whose subtree files become members (deeply nested projects are split out); junk dirs (node_modules, .build, ...) are pruned. Files get byteFingerprint = size:sha256(head4k+tail4k) for name-independent exact-dup detection (shortest path wins). UnitTextComposer builds the embed text: project = name + languages + manifest description + README + listing; collection = name + 40 member titles + snippets.
