How items enter the library. DefaultColdStartScanner walks a granted root: recursive scans run FilesystemDescender (projects/collections become single tagged units, members individually indexed, byte-fingerprint dups indexed but not embedded); shallow scans list the top level. Each candidate is inspected (FileSystemItemInspector), enriched with extracted metadata, indexed into SearchService, embedded into VectorIndex, and recorded in the KnowledgeStore (item + capture event + belongsTo edge) with stable path-derived UUIDs so rescans are idempotent. DefaultDropIntakeHandler covers drag-drop; DefaultOrganizationPipeline + DefaultWorkflowEngine handle organize-mode routing.
