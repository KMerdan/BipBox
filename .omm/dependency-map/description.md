The SPM target graph (Package.swift). Strict inward dependency direction: every implementation target depends only on BipboxCore (the port definitions); BipboxAppSupport composes them; UI and entry points sit on top. BipboxMLX is deliberately a leaf product linked ONLY by BipboxApp so tests, the harness, and all other targets build and run without the heavyweight MLX/Metal dependency chain (mlx-swift-lm 3.31.x, swift-huggingface, swift-transformers).
