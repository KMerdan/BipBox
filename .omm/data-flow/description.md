How a file on disk becomes a node in the semantic graph. A granted folder is scanned recursively; FilesystemDescender decides the unit boundaries (project = one node, collection = one node + searchable members, loose file = itself; exact dups fingerprinted). Every indexed item gets real text content (source code read directly; PDF/doc/image via system extractors/OCR), which feeds both the FTS5 index and the embedding text. Vectors land in the SQLite vector index keyed by model id. Topic discovery runs over discovery SEEDS (aggregate units + capped collection samples + loose docs) and soft-assigns everything else, producing the topics, memberships, and centroid edges the Connections overview renders.
